# Quickstart Walkthrough

This walkthrough explains the small simulated run from the README and the CSV
files it writes.
It focuses on Scenario 2, where the source cohort is deliberately miscalibrated
relative to the target/reference data-generating process.

## Run The Example

From the repository root, run:

```bash
uv run risk-bridge \
  --mode simulated \
  --scenario 2 \
  --nsim 5 \
  --n-target 5000 \
  --n-source 2000 \
  --n-reference 5000 \
  --sample-size 500 \
  --output-root data \
  --run-label quickstart
```

The command writes a directory named like:

```text
data/python_quickstart_<run_id>/
```

`<run_id>` is a timestamp-style identifier for that invocation.
Inside the run directory, Risk Bridge writes `intermediate/` and `final/`.

## Process Steps

1. Generate synthetic cohorts.
   The run creates target, source, and reference cohorts with binary outcome
   `caseY`, discrete features `X1` through `X4`, continuous risk score
   `zOrigin`, and binned risk category `zCat`.

2. Apply Scenario 2 source miscalibration.
   The target and reference cohorts use the target feature and risk model.
   The source cohort keeps the same feature support but uses shifted feature
   probabilities and the Scenario 2 source miscalibration settings recorded in
   `final/run_metadata.csv`.

3. Repeat five simulation iterations.
   `--nsim 5` means each per-iteration output has one row per iteration, except
   sampled-data files and calibration-map files, which have multiple rows per
   iteration.

4. Build two source samples per iteration.
   The PSM path samples source rows using propensity-score matching against the
   target/source pool.
   The RS path samples source rows randomly.

5. Build reference calibration artifacts.
   The reference cohort defines the discrete feature support, maps each feature
   combination into a risk interval, and estimates the external event probability
   for each interval.

6. Fit ML and cMLE models.
   Each iteration fits unconstrained maximum likelihood and constrained maximum
   likelihood on both sampled paths: PSM and RS.

7. Evaluate and export.
   The run writes parameter estimates, threshold metrics, ROC AUC values,
   target prevalence, fit diagnostics, threshold metadata, and run metadata.

## Directory Layout

```text
python_quickstart_<run_id>/
  intermediate/
    pe_by_iter.csv
    sample_psm_by_iter.csv
    sample_rs_by_iter.csv
    xind_by_iter.csv
  final/
    accuracy_base.csv
    accuracy_cml_psm.csv
    accuracy_cml_rs.csv
    accuracy_ml_psm.csv
    accuracy_ml_rs.csv
    accuracy_ref.csv
    est_cml_psm.csv
    est_cml_rs.csv
    est_ml_psm.csv
    est_ml_rs.csv
    fit_diagnostics.csv
    roc_metrics.csv
    run_metadata.csv
    target_prevalence_by_iter.csv
    threshold_ref_fpr.csv
```

Start with `final/run_metadata.csv` to confirm the run settings, then inspect
`final/fit_diagnostics.csv`, `final/est_cml_psm.csv`, and
`final/roc_metrics.csv`.

## Intermediate Files

### `intermediate/sample_psm_by_iter.csv`

Source sample selected by the propensity-score matching path.
For this quickstart, expect `nsim * sample_size` data rows: `5 * 500 = 2500`.

| Column | Meaning | Expected values |
| --- | --- | --- |
| `iter` | Simulation iteration number. | Integers from `1` to `5`. |
| `caseY` | Binary outcome for the sampled source row. | `0` or `1`. |
| `X1` | First discrete feature. | Integer-coded category from the scenario support. |
| `X2` | Second discrete feature. | Integer-coded category from the scenario support. |
| `X3` | Third discrete feature. | Integer-coded category from the scenario support. |
| `X4` | Fourth discrete feature. | Integer-coded category from the scenario support. |
| `zOrigin` | Continuous risk score generated by the Z model. | Floating-point value in `(0, 1]`. |
| `zCat` | Binned category for `zOrigin`. | Integer bin index from `0` through `len(z_bins)`. |

### `intermediate/sample_rs_by_iter.csv`

Source sample selected by random sampling.
It has the same columns and expected row count as `sample_psm_by_iter.csv`.

| Column | Meaning | Expected values |
| --- | --- | --- |
| `iter` | Simulation iteration number. | Integers from `1` to `5`. |
| `caseY` | Binary outcome for the sampled source row. | `0` or `1`. |
| `X1` | First discrete feature. | Integer-coded category from the scenario support. |
| `X2` | Second discrete feature. | Integer-coded category from the scenario support. |
| `X3` | Third discrete feature. | Integer-coded category from the scenario support. |
| `X4` | Fourth discrete feature. | Integer-coded category from the scenario support. |
| `zOrigin` | Continuous risk score generated by the Z model. | Floating-point value in `(0, 1]`. |
| `zCat` | Binned category for `zOrigin`. | Integer bin index from `0` through `len(z_bins)`. |

### `intermediate/xind_by_iter.csv`

Mapping from enumerated feature combinations to reference-data risk intervals.

| Column | Meaning | Expected values |
| --- | --- | --- |
| `iter` | Simulation iteration number. | Integers from `1` to `5`. |
| `x_combination` | One-based identifier for an enumerated `X1`-`X4` feature combination. | Positive integers. |
| `risk_interval` | One-based risk interval assigned to the feature combination. | Positive integers matching intervals in `pe_by_iter.csv`. |

### `intermediate/pe_by_iter.csv`

External calibration probability by reference-data risk interval.

| Column | Meaning | Expected values |
| --- | --- | --- |
| `iter` | Simulation iteration number. | Integers from `1` to `5`. |
| `risk_interval` | One-based reference risk interval. | Positive integers. |
| `p_external` | Estimated external event probability for that interval. | Floating-point probability in `[0, 1]`. |

## Estimate Files

These files share the same columns:

- `final/est_cml_psm.csv`: constrained MLE fit on the PSM sample
- `final/est_cml_rs.csv`: constrained MLE fit on the random source sample
- `final/est_ml_psm.csv`: unconstrained ML fit on the PSM sample
- `final/est_ml_rs.csv`: unconstrained ML fit on the random source sample

Each file has one data row per iteration.

| Column | Meaning | Expected values |
| --- | --- | --- |
| `iter` | Simulation iteration number. | Integers from `1` to `5`. |
| `alpha` | Intercept for the binary outcome risk model. | Floating-point estimate. |
| `beta_X1` | Outcome-model coefficient for `X1`. | Floating-point estimate. |
| `beta_X2` | Outcome-model coefficient for `X2`. | Floating-point estimate. |
| `beta_X3` | Outcome-model coefficient for `X3`. | Floating-point estimate. |
| `beta_X4` | Outcome-model coefficient for `X4`. | Floating-point estimate. |
| `beta_Zcat` | Outcome-model coefficient for binned risk category `zCat`. | Floating-point estimate. |
| `gamma_0` | Intercept for the Z model. | Floating-point estimate. |
| `gamma_X1` | Z-model coefficient for `X1`. | Floating-point estimate. |
| `gamma_X2` | Z-model coefficient for `X2`. | Floating-point estimate. |
| `gamma_X3` | Z-model coefficient for `X3`. | Floating-point estimate. |
| `gamma_X4` | Z-model coefficient for `X4`. | Floating-point estimate. |
| `gamma_sigma` | Scale parameter for the Z model. | Positive floating-point estimate when fitting succeeds. |

## Accuracy Files

These files share the same columns:

- `final/accuracy_cml_psm.csv`: threshold metrics for cMLE on the PSM sample
- `final/accuracy_cml_rs.csv`: threshold metrics for cMLE on the random source sample
- `final/accuracy_ml_psm.csv`: threshold metrics for ML on the PSM sample
- `final/accuracy_ml_rs.csv`: threshold metrics for ML on the random source sample
- `final/accuracy_base.csv`: threshold metrics for the baseline model using `X`
  features only
- `final/accuracy_ref.csv`: threshold metrics for the reference model using `X`
  features and `zCat`

Each file has one data row per iteration.

| Column | Meaning | Expected values |
| --- | --- | --- |
| `iter` | Simulation iteration number. | Integers from `1` to `5`. |
| `TPR` | True positive rate at `threshold_ref_fpr`. | Floating-point value in `[0, 1]`, or `NaN` if no positives are present. |
| `PPV` | Positive predictive value at `threshold_ref_fpr`. | Floating-point value in `[0, 1]`, or `NaN` if no rows are predicted positive. |
| `TNR` | True negative rate at `threshold_ref_fpr`. | Floating-point value in `[0, 1]`, or `NaN` if no negatives are present. |

## Other Final Files

### `final/roc_metrics.csv`

ROC AUC summary by iteration.

| Column | Meaning | Expected values |
| --- | --- | --- |
| `iter` | Simulation iteration number. | Integers from `1` to `5`. |
| `roc_CML_PSM` | ROC AUC for cMLE fit on the PSM sample. | Floating-point value in `[0, 1]`, or `NaN` if the evaluation sample has one class. |
| `roc_CML_RS` | ROC AUC for cMLE fit on the random source sample. | Floating-point value in `[0, 1]`, or `NaN` if the evaluation sample has one class. |
| `roc_ML_PSM` | ROC AUC for ML fit on the PSM sample. | Floating-point value in `[0, 1]`, or `NaN` if the evaluation sample has one class. |
| `roc_ML_RS` | ROC AUC for ML fit on the random source sample. | Floating-point value in `[0, 1]`, or `NaN` if the evaluation sample has one class. |
| `roc_base` | ROC AUC for the baseline model using `X` features only. | Floating-point value in `[0, 1]`, or `NaN` if the evaluation sample has one class. |
| `roc_ref` | ROC AUC for the reference model using `X` features and `zCat`. | Floating-point value in `[0, 1]`, or `NaN` if the evaluation sample has one class. |

### `final/threshold_ref_fpr.csv`

Threshold chosen to make the reference model's false-positive rate closest to
the configured target FPR.

| Column | Meaning | Expected values |
| --- | --- | --- |
| `iter` | Simulation iteration number. | Integers from `1` to `5`. |
| `threshold_ref_fpr` | Score threshold selected for `fpr_target`. | Floating-point threshold in `[0, 1]`. |

### `final/target_prevalence_by_iter.csv`

Observed target outcome prevalence by iteration.

| Column | Meaning | Expected values |
| --- | --- | --- |
| `iter` | Simulation iteration number. | Integers from `1` to `5`. |
| `prevalence_target` | Mean of `caseY` in the generated target cohort. | Floating-point value in `[0, 1]`; near `y_prev` in `run_metadata.csv` for large cohorts. |

### `final/fit_diagnostics.csv`

Solver diagnostics for each fitting path.
Expect two data rows per iteration: one with `path=PSM` and one with `path=RS`.

| Column | Meaning | Expected values |
| --- | --- | --- |
| `iter` | Simulation iteration number. | Integers from `1` to `5`. |
| `path` | Sampling path used for fitting. | `PSM` or `RS`. |
| `mle_success` | Whether unconstrained ML reported success. | Boolean. |
| `cmle_success` | Whether constrained cMLE reported success. | Boolean. |
| `mle_status` | Solver status message for unconstrained ML. | Text from the optimizer. |
| `cmle_status` | Solver status message for constrained cMLE. | Text from the optimizer. |
| `mle_objective` | Final unconstrained negative log-likelihood objective. | Floating-point value. |
| `cmle_objective` | Final constrained negative log-likelihood objective. | Floating-point value. |
| `cmle_max_violation` | Maximum calibration-constraint violation reported by cMLE diagnostics. | Floating-point value; `0` means no positive violation. |

### `final/run_metadata.csv`

One-row run settings and scenario metadata.

| Column | Meaning | Expected values |
| --- | --- | --- |
| `run_id` | Timestamp-style identifier used in the run directory name. | Text. |
| `scenario` | Simulated scenario name. | `Scenario2` for the README quickstart. |
| `nsim` | Number of simulation iterations. | `5`. |
| `Ntarget` | Generated target cohort size. | `5000`. |
| `Nsource` | Generated source cohort size. | `2000`. |
| `Nrs` | Generated reference cohort size. | `5000`. |
| `samplesize` | Source sample size used by each fitting path. | `500`. |
| `y_prev` | Configured target outcome prevalence. | Floating-point probability. |
| `betaz` | Configured outcome-model coefficient for `zCat`. | Floating-point value. |
| `fpr_target` | Target false-positive rate for threshold selection. | Floating-point probability. |
| `alpha` | Configured target outcome-model intercept. | Floating-point value. |
| `parameterOR` | Semicolon-separated configured outcome-model coefficients. | Text list of floating-point values. |
| `gamma` | Semicolon-separated configured Z-model starting values. | Text list of floating-point values. |
| `seed` | Random seed used by the run. | Non-negative integer. |
| `miscalibration_a` | Scenario source miscalibration additive term. | Floating-point value. |
| `miscalibration_b` | Scenario source miscalibration multiplicative term. | Floating-point value. |
| `source_miscalibration_a` | Explicit source miscalibration additive term. | Floating-point value; same as `miscalibration_a` in this output. |
| `source_miscalibration_b` | Explicit source miscalibration multiplicative term. | Floating-point value; same as `miscalibration_b` in this output. |
| `target_feature_specs` | Serialized target/reference feature support and generation settings. | JSON text. |
| `source_feature_specs` | Serialized source feature support and generation settings. | JSON text; differs from target specs in Scenario 2. |
| `calibration_tolerance` | Allowed slack for calibration inequalities. | Non-negative floating-point value. |
| `n_jobs` | Process-level iteration parallelism. | Positive integer. |
| `path_jobs` | Thread-level PSM/RS fitting parallelism inside each iteration. | Positive integer. |
| `intermediate_flush_every` | Number of iterations between intermediate CSV flushes. | Positive integer. |

## Reading Results

Use `fit_diagnostics.csv` before comparing estimates.
If a solver did not succeed or has a positive `cmle_max_violation`, treat the
corresponding estimate and metric rows cautiously.

Compare PSM and RS paths to see how sampling changes estimates and metrics.
Compare ML and cMLE files to see the effect of adding calibration constraints.
Use `accuracy_ref.csv`, `accuracy_base.csv`, and `roc_metrics.csv` as reference
points rather than as fitted Risk Bridge outputs.
